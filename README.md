# Alek's Design System

A personal design system built from scratch — not a template, 
not a fork.

## Token architecture
Structured across two layers:

**Primitive tokens** — raw scale values. Colors, spacing, 
typography, sizing. No semantic meaning, just the raw material.

**Semantic tokens** — purpose-named, following category > 
subcategory > component > role > state naming convention. 
Every token has both light and dark values defined.

## Palette
- **Primary:** Indigo scale (100–1000)
- **Brand accent:** Amber scale (100–800)  
- **Neutrals:** Cool gray scale (100–1000)
- **Status:** Success, error, warning, info
